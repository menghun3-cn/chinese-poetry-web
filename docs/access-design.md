# 访问设计文档

## 信息架构

- 顶部导航：品牌、阅读入口、数据入口。
- 阅读页：站点说明、数据状态、统计卡片、筛选区、作品列表、正文阅读器。
- 数据页：索引说明、统计卡片、文集规模图表、热门作者、收藏夹。

## 用户路径

1. 用户进入首页，先看到站点定位和当前索引规模。
2. 用户通过关键词、文集、作者或标签缩小范围。
3. 用户在左侧结果列表选择作品，右侧阅读正文。
4. 用户可复制全文或收藏作品。
5. 用户进入数据页查看索引构成，并回到阅读页继续检索。

## 响应式策略

- 375px：筛选项单列排列，列表和阅读器纵向堆叠，按钮触控高度不小于 44px。
- 768px：筛选区保持清晰分组，结果列表和阅读器仍可纵向阅读。
- 1440px：结果列表固定在左侧，阅读器占据主阅读空间。

## 组件状态

- 数据加载：显示加载说明，避免空白页面。
- 加载失败：显示可恢复错误说明，提示执行构建或检查静态 JSON。
- 空结果：提供清空筛选动作。
- 复制成功：在阅读器内显示短暂状态反馈。
- 收藏状态：按钮区分“收藏”和“已藏”。

## 视觉设计

- 背景使用米纸色，正文区域使用轻纹理纸张感，降低长时间阅读疲劳。
- 主色使用墨绿色，强调搜索、选中和主操作。
- 赭色用于标签和文化语境提示，避免全站单一色相。
- 组件圆角控制在 8-12px，边框优先于重阴影。

## 可访问性

- 搜索、作者、文集筛选均提供可见 label。
- 顶部导航具备 `aria-label`。
- 错误状态使用 `role="alert"`，加载和空状态使用 `role="status"`。
- 交互元素保留 `focus-visible` 焦点环。

## 部署访问

GitHub Action 会在 `main` 分支推送时发布到 Cloudflare Pages。需要在仓库 Secrets 配置：

- `CLOUDFLARE_API_TOKEN`
- `CLOUDFLARE_ACCOUNT_ID`

Cloudflare Pages 项目名默认为 `chinese-poetry-web`。
