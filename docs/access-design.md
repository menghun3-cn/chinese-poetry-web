# 访问设计文档

## 信息架构

- 顶部导航：品牌、阅读入口、数据入口。
- 阅读页：阅读工作台、快速检索、当前阅读摘要、辅助数据概览、篇目列表、正文阅读器。
- 数据页：索引说明、统计卡片、文集规模图表、热门作者、收藏夹。

## 用户路径

1. 用户进入首页，优先看到关键词检索、文集筛选、作者筛选和热门入口。
2. 用户通过关键词、文集、作者、标签或常读作者缩小范围。
3. 用户在篇目列表选择作品，阅读摘要和正文阅读器同步更新。
4. 用户可在当前阅读区直接进入正文，也可在阅读器内复制全文或收藏作品。
5. 用户进入数据页查看索引构成，并回到阅读页继续检索。

## 响应式策略

- 375px：首屏纵向展示检索、当前阅读摘要、篇目和正文，按钮触控高度不小于 44px。
- 768px：检索区保持清晰分组，热门标签和常读作者横向滚动，列表和阅读器纵向阅读。
- 1440px：篇目列表固定在左侧宽栏，阅读器占据主阅读空间，当前阅读摘要保留在首屏右侧。

## 组件状态

- 数据加载：显示加载说明，避免空白页面。
- 加载失败：显示可恢复错误说明，提示执行构建或检查静态 JSON。
- 空结果：提供清空筛选动作。
- 复制成功：在阅读器内显示短暂状态反馈。
- 收藏状态：按钮区分“收藏”和“已收藏”，篇目列表显示已收藏标记。

## 视觉设计

- 背景使用米纸色和克制网格纹理，正文区域使用轻纹理纸张感，降低长时间阅读疲劳。
- 主色使用墨绿色，强调搜索、选中和主操作。
- 赭色用于标签和文化语境提示，避免全站单一色相。
- 页面工具和重复项圆角控制在 8px，边框优先于重阴影。

## 可访问性

- 搜索、作者、文集筛选均提供可见 label。
- 顶部导航具备 `aria-label`。
- 错误状态使用 `role="alert"`，加载和空状态使用 `role="status"`。
- 交互元素保留 `focus-visible` 焦点环。

## 部署访问

GitHub Action 会在 `main` 分支推送时发布到 Cloudflare Pages。需要在仓库 Secrets 配置：

- `CLOUDFLARE_API_TOKEN`
- `CLOUDFLARE_ACCOUNT_ID`

Cloudflare Pages 项目名默认为 `chinese-poetry-web`。
